Deliverable 02 · Primitives

The component library.

Ten primitives, every variant. Both modes side by side, drawing only on the tokens defined in deliverable 01. No new colors, no new font sizes, no new radii.

01 Button

Three variants: primary (terracotta, single per view), secondary (hairline border on surface), ghost (no chrome until hover). Sizes scale by height; radius stays geometric. Focus is a 3px soft-accent ring, never a glow.

.btn / .btn--primary / .btn--secondary / .btn--ghost / .btn--sm / .btn--lg

02 Input

Text, password, select. Default border is border-strong so the field reads as input without a label, hover lifts to fg-faint, focus is accent with the soft ring. Invalid uses danger; never red 500. Disabled fills with subtle.

08 Output block

Where streamed text lands. Sunken surface and a hairline border so the eye knows the model wrote this, not the user. A pulsing accent dot when live; a faint dot when complete. No skeleton loader theater, no jumbo spinner. The caret is a single terracotta block.

09 Post block

The one editorial moment per screen. When the model returns a draft wrapped in <post> tags, it renders as Source Serif on surface, set off by a 3px terracotta left rule. The serif signals "this is the artifact you'd publish," not chat output.

.post / .post-meta

Draft · 184 words

Last week a portfolio CEO asked me how we decide what to ship. I gave the answer I always give, and watched it not land.

The honest version is that we don't decide. We notice what is already moving and we clear the path.

Three things I keep returning to when I'm trying to teach this:

One. The team is already telling you what wants to move. Your job isn't to pick the winner. It's to hear which thing keeps coming up in the standups, the side-channel DMs, the offhand comments in product review. That's the candidate.
